# 要件確認質問

## Q1: 対象タスクの絞り込み
スケジューリング対象タスクはどう絞るか？
- a) 特定のプロジェクト ID を指定する（現在 `main.py` にハードコード済みの `6gc6X66RQH7Qcmxh`）
- b) 今日の期限タスクすべて
- c) プロジェクト・ラベル等を実行時引数で指定
A. a

## Q2: 作業可能時間帯
今日の作業開始・終了時刻をどう決めるか？
- a) スクリプト引数で渡す（例: `--start 09:00 --end 18:00`）
- b) 固定値をコード内に定義（例: 9:00〜18:00）
- c) 別の設定ファイルで管理
A. a

## Q3: 作業開始時刻（ベース）
最初のタスクを何時から割り当てるか？（例: 現在時刻の次の00/30分区切り、または固定で「今日の9:00」）
A. スクリプト引数で指定した開始時間から15分区切り

## Q4: 休憩・ランチ時間
昼休みなどの除外時間帯は考慮するか？
- a) 考慮しない（詰めてスケジュール）
- b) ランチ休憩（例: 12:00〜13:00）を除外する
A. 考慮する。これもスクリプト引数で設定する --lunch-start 12:00 --lunch-end 13:00

## Q5: タスク優先度の扱い
Todoist の priority（p1〜p4）はスケジューリングに影響するか？
- a) 優先度が高いタスクを先に配置する
- b) 優先度は無視して description の順序（または取得順）で配置
A. b

## Q6: 見積時間がないタスクの扱い
description に数値が入っていないタスク（空白・テキストのみ）はどう扱うか？
- a) スキップする（dueString を変更しない）
- b) デフォルト時間（例: 1.0h）を割り当てる
- c) エラーとして報告する
A. c

## Q7: タスクが作業可能時間に収まらない場合
全タスクの合計見積時間が作業可能時間を超えた場合の挙動は？
- a) 収まる範囲でスケジュールし、収まらないタスクは未設定のままにする
- b) 翌日以降に割り当てる
- c) ユーザーに警告を表示する
A. a

## Q8: 時間の粒度
タスク開始時刻の粒度は？（例: 15分刻み、30分刻み、見積時間どおり）
A. 15分刻み

## Q9: reschedule-tasks vs update-tasks
日付更新には `reschedule-tasks` を使う仕様だが（繰り返しタスクの保護のため）、このスケジューラでも同様に `reschedule-tasks` で統一するか？
- a) はい（安全側）
- b) `update-tasks` で dueString に時刻付き文字列を渡す（繰り返しタスクが混在する可能性を許容）
A. b
