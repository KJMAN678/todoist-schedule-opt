# 要件定義書: Todoist タスク最適スケジューリング

## 概要

Todoist の特定プロジェクトからタスクを取得し、各タスクの見積作業時間（`description` フィールドに数値で記録）と作業可能時間帯をもとに最適スケジュールを計算して、`dueString` に実行開始時刻を設定する Python スクリプト。

---

## 機能要件

### FR-1: タスク取得
- 指定したプロジェクト ID（スクリプト引数 `--project-id`）のタスクを Todoist MCP API から取得する
- デフォルト値: `6gc6X66RQH7Qcmxh`（現行 `main.py` のプロジェクト ID）

### FR-2: 見積時間のパース
- 各タスクの `description` フィールドから数値（時間単位、例: `0.5`）をパースする
- `description` が空、または数値として解釈できない場合はエラーとして stderr に報告し、そのタスクをスケジュール対象から除外する

### FR-3: スケジュール最適化
- 入力として以下を受け取る
  - 作業開始時刻（`--start`、例: `09:00`）
  - 作業終了時刻（`--end`、例: `18:00`）
  - ランチ開始（`--lunch-start`、例: `12:00`）
  - ランチ終了（`--lunch-end`、例: `13:00`）
- 作業可能スロット（開始〜終了のうちランチを除いた時間帯）を 15 分スロット単位で管理する
- タスクを取得順に並べ、スロットが空いている限り順に割り当てる
- ortools（CP-SAT ソルバー）を使って最適スロット割り当てを計算する
- 合計見積時間が作業可能時間を超えた場合、入りきらないタスクはスケジュール対象外とし、その旨を stderr に警告する

### FR-4: dueString の更新
- スケジュール済みタスクに対し、`update-tasks` ツールで `dueString` を `YYYY-MM-DD HH:MM` 形式で設定する
- 未スケジュールタスク（見積エラー・時間超過）の `dueString` は変更しない

### FR-5: 実行レポート
- 実行後に標準出力でスケジュール結果を表示する
  - タスク名・見積時間・割り当て開始時刻
  - スキップしたタスクと理由

---

## 非機能要件

### NFR-1: 実行環境
- Python 3.13、`uv run` で実行
- 依存: `mcp==1.27.0`、`ortools==9.15.6755`（追加インストール不要、既存依存）

### NFR-2: 認証
- `TODOIST_API_TOKEN` 環境変数を使用（`.env.local` で管理）

### NFR-3: CLI インターフェース
- `argparse` で以下の引数を受け付ける

| 引数 | 必須 | デフォルト | 説明 |
|------|------|-----------|------|
| `--project-id` | 任意 | `6gc6X66RQH7Qcmxh` | 対象プロジェクト ID |
| `--start` | 必須 | なし | 作業開始時刻（HH:MM） |
| `--end` | 必須 | なし | 作業終了時刻（HH:MM） |
| `--lunch-start` | 任意 | なし | ランチ開始時刻（HH:MM） |
| `--lunch-end` | 任意 | なし | ランチ終了時刻（HH:MM） |
| `--date` | 任意 | 今日の日付 | スケジュール対象日（YYYY-MM-DD） |

### NFR-4: エラーハンドリング
- `description` が数値でないタスクは stderr に報告して続行（プロセスは終了しない）
- API エラーは例外を再送出してスタックトレースを表示

---

## 制約・前提

- タスクの実行順序は取得順（優先度は考慮しない）
- 時刻粒度は 15 分刻み（タスク見積時間は 15 分単位に切り上げ）
- `update-tasks` を使用（`reschedule-tasks` は使わない）
- 繰り返しタスクへの影響は許容する
